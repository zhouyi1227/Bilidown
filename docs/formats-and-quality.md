# 画质、编码与音频格式

Bilidown 只显示 Bilibili 对当前投稿、分 P 和登录态实际返回的格式。相同分辨率可能有多种编码、码率和动态范围，因此不会再合并为一个模糊的“1080P”。

## 兼容 MP4

默认模式只列出 H.264 SDR 视频，并搭配最佳 AAC 音频，无重编码封装为 MP4。它通常能在 Windows、macOS、手机、电视和剪辑软件中直接播放。

同高度的“1080P 高清”和“1080P 高码率”会分别显示；选择项还包含帧率和实际码率。若登录态只返回普通版本，应用不会制造高码率选项。

## 原始质量

该模式展示所有源视频流，包括：

- H.264：兼容性最好，文件通常较大；
- HEVC/H.265：压缩效率更高，旧设备可能无法硬件解码；
- AV1：效率高，但播放器和剪辑软件兼容性取决于设备；
- HDR10 / Dolby Vision：需要相应屏幕、系统和播放器。

应用按精确格式 ID 下载，不转码视频。FFmpeg 根据流兼容性选择 MP4 或 MKV；MKV 并不代表画质较低。多 P 下载只显示所有已选分 P 共同拥有的精确变体。

## 4K 与会员画质

只有解析结果包含 2160P 时才显示 4K。`BV1ACNJ6VEwP` 的源上限是 1080P；`BV1NGZtBwELa` 在有效大会员登录态下可用于验证 4K、高码率和 Dolby Vision。游客解析后只有 1080P 并非程序故障。

## 音频

- **AAC / M4A 源流**：默认选项，不转码，兼容性最好。
- **最佳源流**：优先保存真实存在的 FLAC 或 Dolby 音频，否则选择最高码率源流；不会把有损音频伪装成 FLAC。
- **MP3 VBR V2**：使用 FFmpeg/LAME 转码，适合需要 MP3 的设备，会产生一次有损转换。

封面直接保存元数据中的可信原图，不缩放、不重新压缩。若封面地址不属于受信任的 Bilibili/hdslb 域，预览会留空，但不会阻止视频或音频解析。

## 批量画质回退

收藏夹、番剧等批量资源按每个条目单独选择格式。应用优先下载不超过所选高度的 H.264 + AAC 兼容组合；若某一项没有该高度或编码，会只对该项向下回退，不影响其他项。

## 字幕与弹幕

- 普通字幕由 Bilibili/yt-dlp 实际返回，部分字幕需要登录。
- 弹幕 XML 是 Bilibili 原始 `danmaku` 字幕轨，适合归档和后续处理。
- 弹幕 ASS 由 Bilidown 本地转换，可在常见播放器叠加。滚动、顶部和底部弹幕受支持；高级定位、脚本弹幕和互动点击路径不能完整复现。
