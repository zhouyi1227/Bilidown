# 安全与隐私

## 本地边界

Bilidown sidecar 只监听随机的 `127.0.0.1` 端口，不允许局域网访问。启动时生成随机会话令牌；API 同时校验令牌和精确 Tauri Origin。严格 CSP 禁止第三方脚本、字体和 CDN。

应用会连接 Bilibili 页面、API 和媒体 CDN，也可能访问 `b23.tv` 解析短链。短链仅允许 HTTPS、限制跳转次数和响应大小，并要求最终域名属于 Bilibili。

## Cookie 与账户资料

推荐登录窗口使用独立隐私 WebView，读取包括 HttpOnly 在内的 Bilibili Cookie 后通过受控命令直接上传到内存会话。浏览器直读和 Netscape 文件同样只保留 Bilibili 域条目。普通前端 API 永远不会返回完整 Cookie、`SESSDATA`、UID 或媒体签名 URL。

应用不保存登录态、历史任务或遥测。退出后任务状态和内存 Cookie 消失。已完成的媒体文件保留在用户选择的输出目录。

## 文件与日志

下载使用任务专属临时目录，完成后再无覆盖移动到输出目录；取消或失败会清理 `.part` 和任务临时文件。直播只有在用户明确“停止并保留”时才会把当前 MPEG-TS `.part` 转为 `.ts`，否则清理。文件名会过滤 Windows/macOS 不安全字符。

错误日志经过脱敏，不应包含 Cookie 或完整签名 URL。日志仍可能包含视频 BV 号、错误类别和部分路径，因此公开前应人工检查。

## 使用边界

Bilidown 不绕过 DRM、会员、付费、地区或账户限制。使用者负责确认保存权限。不要用它抓取他人隐私内容、自动化批量请求或规避 Bilibili 控制。

发现令牌绕过、Cookie 泄漏、任意文件写入、短链 SSRF 或签名 URL 泄漏时，请按[安全政策](../SECURITY.md)私密报告，不要先公开利用细节。
